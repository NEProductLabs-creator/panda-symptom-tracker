The design system at /styles/design-system.css has been updated with a new warm earth palette. The authenticated app screens (login, signup, dashboard, symptom log entry forms, timeline / history views, child profile pages, reports / export views, top navigation, empty states) were styled previously using the old tokens. They need to be retuned to use the new tokens, without changing any layout, structure, copy, or functionality.

Apply these token replacements across all authenticated routes only. Do not touch the landing page, which intentionally retains the old palette.

Background and surface replacements:
- var(--cream) becomes var(--bg-app)
- var(--cream-warm) becomes var(--bg-card) for cards, modals, and elevated surfaces
- var(--cream-warm) becomes var(--bg-subtle) for low-emphasis accent zones
- var(--rule) becomes var(--rule-soft) for most dividers, kept as var(--rule) only where a stronger separator is intentional

Text replacements:
- var(--ink) stays as var(--ink) but should now resolve to the new warmer brown
- var(--ink-soft) stays as var(--ink-soft), now also warmer

Primary action color shift (this is the most important change):
- Anywhere forest green was used as the primary button background (var(--forest), var(--forest-deep)), replace with var(--clay) and var(--clay-deep) for hover
- Forest green should no longer appear as a primary action color anywhere in the authenticated app
- Active route indicators in the top nav: the 1px underline should now be var(--clay), not forest
- Form input focus states: the border on focus should now be var(--clay), with a subtle var(--clay-wash) background tint

Accent replacements:
- The old var(--terracotta) accent role is now filled by var(--clay), which is the primary brand color rather than a secondary accent
- Where a deep, grounded color is still needed (dark text emphasis, the user menu, deep chart series, header bars), use var(--cocoa) instead of var(--forest-deep)
- Severity dots and intensity scales should now use the five-step clay gradient defined in the design system: var(--clay-wash), var(--clay-soft), #d68866, var(--clay), var(--clay-deep)
- Calm day markers and positive indicators should use var(--mist)

Specific component updates:

Login and signup pages:
- Card background changes from var(--cream-warm) to var(--bg-card), pure white
- Page background remains warm, now var(--bg-app)

Top navigation in authenticated views:
