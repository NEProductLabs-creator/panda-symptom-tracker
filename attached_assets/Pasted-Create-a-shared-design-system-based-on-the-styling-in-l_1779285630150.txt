Create a shared design system based on the styling in landing.html. The goal is for the rest of the app to feel like the same product as the landing page.

Create a file at /styles/design-system.css (or the equivalent for this project's structure) containing:

1. CSS custom properties (root variables) for the full color palette:
   --cream: #f5efe4
   --cream-warm: #ede4d3
   --ink: #1f2a26
   --ink-soft: #3a4742
   --forest: #2d4a3e
   --forest-deep: #1a2f26
   --terracotta: #c4623d
   --terracotta-soft: #d68866
   --sage: #8a9a87
   --gold: #b08846
   --rule: #d4c8b0

2. Google Fonts imports for Fraunces (display) and Newsreader (body), with the same weights and optical sizing used on the landing page.

3. Base typography rules:
   - Body default: Newsreader, 16-18px, line-height 1.65, color var(--ink), background var(--cream)
   - Headings: Fraunces, weight 350-500, letter-spacing -0.015em to -0.025em, color var(--forest-deep)
   - Italics in headings should use color var(--terracotta) as an accent

4. Reusable button classes:
   - .btn-primary: forest background, cream text, 18px 36px padding, uppercase 15px letter-spaced label, hover turns terracotta and lifts 1px
   - .btn-secondary: transparent background, forest border, forest text, same padding, hover fills forest
   - .btn-text: italic underlined text link, terracotta on hover

5. Form input styles:
   - Cream-warm background, 1px solid rule border, 14-16px padding, Newsreader font
   - Focus state: 1px solid terracotta border, no harsh blue outlines
   - Labels in Fraunces italic, var(--ink-soft), 14px, letter-spacing 0.05em

6. Card / container styles:
   - .panel: cream background with 1px solid rule borders, subtle shadow, 4px border radius
   - Generous interior padding (32-48px on desktop)

7. Section header pattern (label + heading) from the landing page, reusable.

Import this design system globally so all routes inherit it. Remove or override any existing global styles that conflict (especially generic blues, purples, system fonts, or default Bootstrap/Tailwind defaults).

Do not change any app functionality. Only update styling.
